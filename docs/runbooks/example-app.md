# Example App Runbook

## Prerequisites

- **Android**: JDK 17, Android SDK (API 34), Android Studio
- **iOS**: Xcode 15+, CocoaPods (`gem install cocoapods`)
- **Both**: Gradle 8.x (wrapper included)

## Quick Start

```bash
make demo-ios       # Setup + build → prints Xcode instructions
make demo-android   # Setup + build → prints Android Studio instructions
make demo-all       # Both platforms
```

Each target installs dependencies, builds, then prints instructions for running in the native IDE.

## How Dependencies Resolve

Both demos resolve native SDK versions from **published artifacts** pinned in `gradle/libs.versions.toml`:

- **Android**: `com.encorekit:encore` → Maven Central
- **iOS**: `EncoreKit` → CocoaPods trunk

To bump native SDK versions, edit `gradle/libs.versions.toml`:
```toml
encore-android = "1.4.27"
encore-ios = "1.4.39"
```

Then re-run `make demo-all`.

## Smoke Test Checklist

- [ ] App launches without crash
- [ ] `configure()` completes (check debug logs)
- [ ] `identify()` sends user data (check network tab or logs)
- [ ] `placement("demo").show()` displays offer UI
- [ ] Dismissing offer returns `NotGranted(USER_CLOSED)`
- [ ] `reset()` clears user state
- [ ] Purchase flow triggers `onPurchaseRequest` handler

## Makefile Targets

| Target | Description |
|:-------|:------------|
| `make demo-ios` | Setup + build iOS, print Xcode instructions |
| `make demo-android` | Setup + build Android, print AS instructions |
| `make demo-all` | Both platforms |
| `make setup` | Install dependencies only (no build) |
| `make clean-example` | Remove build artifacts |

## Troubleshooting

### `No such module 'EncoreKMP'` (Xcode)

Run `make demo-ios` or `make setup` first. The KMP framework is generated by `pod install`.

### Pod install fails with version error

Check that `gradle/libs.versions.toml` has a valid `encore-ios` version and that the version exists on CocoaPods trunk.
